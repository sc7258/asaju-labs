## 2026-04-05
- 달력 입력은 `양력 / 음력 / 윤달` 단일 선택으로 노출하고, 내부적으로 `calendarType` / `isLeapMonth`로 변환한다.
- 생년월일시는 화면에서 `YYYY MMDD` 또는 `YYYY MMDD HHmm` 형식으로 보여주고, URL과 공유 값은 공백 없는 숫자로 유지한다.
- 시간이 없으면 화면상 시주는 `?`로 표시하고, 계산은 `12:00` 기준 근사값으로 처리한다.
- 상단 액션은 긴 텍스트 버튼보다 compact한 아이콘 버튼을 우선한다.
- 공유 버튼은 현재 주소를 그대로 복사하지 않고, 폼의 최신 입력값으로 URL을 다시 조합해 공유한다.
- 공유 미리보기는 정적 카드 대신 현재 입력값으로 동적으로 생성한 OG 이미지(`/api/og`)를 사용한다.
- 로컬 개발 환경이나 사설망 주소에서는 메신저 미리보기가 가능하도록 공개 주소(`NEXT_PUBLIC_SITE_URL` 또는 `https://sajucube.vercel.app`)를 공유 기준 URL로 사용한다.
- 공유용 만세력 이미지는 부가 설명보다 보드 자체의 판독성을 우선하고, OG 이미지에서는 `사주 + 천을귀인/공망 + 대운` 중심으로 구성해 흰 여백을 줄인다.
- Vercel 프로젝트 이름은 `sajucube`를 사용한다.
- 배포는 전역 CLI가 아니라 프로젝트 로컬 CLI 기준으로 `npm run deploy:prod`, `npm run deploy:prod:logs`를 사용한다.
- 만세력 계산 라이브러리는 `@gracefullight/saju`를 사용한다.
- 테스트는 Vitest로 구성한다.
- PWA는 추가 의존성 없이 Next.js App Router 파일 규칙(`manifest.ts`, `icon.tsx`, `apple-icon.tsx`)과 로컬 서비스 워커(`public/sw.js`) 조합으로 구성한다.
- 서비스 워커는 캐시로 인한 개발 혼선을 막기 위해 프로덕션에서만 등록한다.
- 사용자에게 보이는 앱명과 공유 표기는 모두 `SajuCube` 혼합 대소문자 표기를 사용한다.
- 앱 버전은 `package.json`의 `version`을 단일 기준으로 관리하고, 앱 화면에서는 상시 노출하지 않으며 `앱 정보` UI에서만 확인할 수 있게 한다.
- 배포 직전에는 `package.json`의 patch 버전을 기본적으로 1씩 올리고, minor 버전 증가는 명시 요청이 있을 때만 진행한다.
- 공유 OG 이미지는 만세력 보드를 중심으로 유지하되, 설치 아이콘과 연결해 인지할 수 있도록 좌상단에 작은 브랜드 배지와 4색 큐브 마크를 넣는다.
- PWA 설치 아이콘 갱신이 잘 반영되도록 manifest 링크와 manifest 내부 icon 경로에 빌드별 버전 쿼리를 붙인다.
- 서비스 워커는 `manifest.webmanifest`, `icon`, `apple-icon`, `favicon.ico`를 캐시하지 않도록 두고, `sw.js`와 manifest는 no-cache 헤더로 최신 버전을 우선 받게 한다.
- 만세력 보드 좌측 빈 영역에는 `성별 + 이름`과 변환된 `양 YYYY-MM-DD HH:mm` 요약을 배치하고, 음력 입력이어도 표시 기준은 항상 양력으로 맞춘다.
- 만세력 보드 좌측 출생 정보 요약에는 양력과 음력 생일을 함께 표시하고, 윤달 입력으로 양력 요약을 표시할 때는 접두를 `양` 대신 `윤`으로 표기한다.
- 만세력 보드의 보조 정보(기둥 위치, 적중 위치, 대운/세운 머리말), 대운/세운 구분선, 사주판 배경색은 기본으로 노출/적용하지 않고, 공유 버튼 옆 설정 버튼에서 선택적으로 켤 수 있게 한다.
- 입력 바의 `표시 설정` 버튼은 당분간 숨기고, 상단 액션은 생성/공유/앱 정보 중심으로 단순화한다.
- 생년월일시 입력은 타이핑 도중에도 `YYYYMMDD`, `YYYYMMDDHH`, `YYYYMMDDHHmm` 길이에서 자동 제출해 결과 보드가 바로 따라오게 한다.
- `YYYYMMDDHH`까지만 입력된 경우는 분을 `00`으로 보정해 즉시 계산한다.
- 짧은 시간 안의 연속 자동 반영은 새 히스토리를 계속 추가하지 않고 현재 엔트리를 덮어써서, `뒤로 가기` 시 중간 입력 상태보다 이전에 보던 명식으로 돌아가게 한다.
- 자동 반영은 전체 페이지 재로딩 대신 App Router의 클라이언트 라우팅으로 처리하고, 편집 중에는 날짜/시 자동 반영 응답이 돌아와도 입력 중인 초안 값을 덮어쓰지 않게 한다.
- 모바일 키보드 접기의 `blur` 이벤트는 기기마다 불안정하므로 별도 확정 동작을 싣지 않고, 대신 입력창이 들고 있는 최신 draft 생년월일시를 차트가 직접 참조해 draft와 다른 오래된 자동 반영 응답은 화면에 반영하지 않게 한다.
- 검색 파라미터가 없는 첫 진입에서는 고정 샘플 명식 대신 서울 현재 날짜와 시간을 기본 입력값으로 사용한다.

- 일반 만세력의 기본 만세력은 차샘만세력 기준 `본원`으로 본다.
- 차샘만세력의 출생 정보는 입력이 양력/음력 어느 쪽이든 표시 시에는 양력 생일 기준으로 맞추고, 윤달 입력은 `윤 YYYY-MM-DD` 형식으로 표기한다.

## 2026-04-06 차샘만세력 규칙
- 음력 입력과 차샘만세력 날짜 체인의 양음력 변환은 `@gracefullight/saju`의 변환 헬퍼 대신 `korean-lunar-calendar`를 사용해 한국천문연구원 기준으로 맞춘다.
- 차샘만세력의 날짜 체인 출발점은 `solar` 입력이면 사용자가 입력한 `YYYY-MM-DD` 숫자를 그대로 `본원`으로 쓰고, `lunar` 입력이면 한국 기준으로 변환된 실제 양력 생일을 `본원`으로 사용한다.
- 차력은 본원의 양력 생년월일 숫자를 같은 해의 음력 생년월일 숫자로 치환한 만세력으로 정의하고, 시간은 본원과 동일하게 유지한다.
- 차력 목표 음력 날짜가 실제로 존재하면 그 날짜를 그대로 사용한다.
- 차력 목표 음력 날짜가 존재하지 않으면 바로 이전의 실제 음력 날짜와 바로 다음의 실제 음력 날짜를 후보로 잡는다.
- 없는 음력 날짜의 보정은 오전/오후 규칙만으로 고정하지 않고, 두 후보를 같은 시각으로 양력 변환했을 때 본원의 실제 양력 생년월일시와 더 가까운 쪽을 우선 선택한다.
- 두 후보의 양력 실제 거리가 같을 때만 보조 규칙으로 `12시 전이면 이전일`, `12시 이후면 다음일`을 적용한다.
- 따라서 `1/30`이 없는 달의 오후 출생처럼 다음 후보가 `2/1`이 되어 양력 기준으로 더 멀어지면 이전 후보를 선택할 수 있고, 반대로 `1/31` 오전이라도 다음 후보가 양력 기준으로 더 가까우면 다음 후보를 선택할 수 있다.
- 허자 본원은 차력에서 확정된 음력 생년월일 숫자를 다시 양력 생년월일 숫자로 치환한 뒤, 차력 계산과 같은 방식으로 처리한 만세력으로 정의한다.
- 허자 차력은 허자 본원의 양력 생년월일 숫자를 다시 같은 해의 음력 생년월일 숫자로 치환한 만세력으로 정의한다.
- 예시로 본원이 `양 1972-01-26`이면 차력은 `음 1972-01-26`에 대응하는 날짜이고, 그 결과가 `양 1972-03-11`이라면 허자 본원은 `음 1972-03-11`을 같은 규칙으로 처리한 결과인 `양 1972-04-24`, 허자 차력은 다시 `음 1972-04-24`이 된다.
- 부허자 계열은 허자 계열과 반대 방향으로 계산하고, 본원보다 앞쪽 날짜 체인으로 본다.
- 부허자 차력은 본원의 양력 날짜를 먼저 실제 음력 날짜로 변환한 뒤, 그 음력 생년월일 숫자를 다시 양력 생년월일 숫자로 놓은 만세력으로 정의한다.
- 부허자 본원은 부허자 차력에서 나온 양력 날짜를 다시 실제 음력 날짜로 변환하고, 그 음력 생년월일 숫자를 다시 양력 생년월일 숫자로 놓은 만세력으로 정의한다.
- 예시로 본원이 `양 1972-01-26 (음 1971-12-11)`이면 부허자 차력은 `양 1971-12-11 (음 1971-10-24)`, 부허자 본원은 `양 1971-10-24 (음 1971-09-06)`이 된다.
- 부허자 단계에서 `1999-02-29`처럼 실제로 존재하지 않는 양력 숫자가 나오면, 먼저 `전날 -> 전전날 -> 다음날 -> 다다음날`의 실제 양력 후보 중 음력 날짜가 목표 음력 숫자와 정확히 같은 날이 있는지 확인한다.
- 그런 후보가 없으면 부허자 보정은 `12시 전이면 전날`, `12시 이후면 다음날` 규칙으로 처리한다.
- 차샘만세력 화면은 6개 독립 만세력을 `부허자 2판 -> 본원/차력 2판 -> 허자 2판` 순서의 세로 그룹으로 배치한다.
- 각 판은 기본적으로 접힌 상태로 시작하고, 접힌 상태에서는 사주 8자 중심의 상단 보드만 보여주고 대운/세운은 숨긴다.
- 펼친 판만 해당 만세력의 대운/세운을 보여주고, 판별 라벨은 화면에 크게 노출하지 않는다.
- 차샘만세력 공유 OG는 추후 2행 3열 배치로 6개 판의 사주 8자만 보여주고, 설명 라벨은 생략하는 방향을 기준으로 잡는다.
- 차샘만세력 OG 타일의 색상 계산은 글자를 다시 천간/지지로 해석하지 않고 `viewModel`의 `stemElement`/`branchElement`를 우선 사용하며, 시간 미입력으로 `?` 시주가 들어오면 중립 타일로 처리해 이미지 생성이 깨지지 않게 한다.
- 차샘만세력 OG는 판독성을 우선해 `본원/차력` 2판만 4주 전체를 보여주고, `부허자/허자` 4판은 `일주/월주`만 표시한다.

## 2026-04-12
- Charyeok fallback for missing lunar dates is based on nominal date closeness, not on the converted solar date distance.
- Example: for `1973-12-31`, prefer lunar `1974-01-01` over lunar `1973-12-29` because nominal solar `1974-01-01` is closer to `1973-12-31`.
