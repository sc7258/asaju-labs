# 계획

## 현재 목표
- 1차 목표는 입력값을 받아 만세력 화면을 정확하고 compact하게 보여주는 MVP를 완성하는 것이다.
- 참고 이미지와 최대한 비슷한 밀도, 배치, 색감으로 맞춘다.

## 큰 단계
1. 입력 → 계산 → 결과 표시 흐름을 end-to-end로 완성한다.
2. 만세력 보드 UI를 레퍼런스에 가깝게 다듬는다.
3. 테스트로 입력 파싱, 날짜 변환, 대운/세운 표시 규칙을 고정한다.
4. 이후 차샘만세력 확장 기능으로 넘어간다.

## 현재 작업
- 입력 → 계산 → 결과 표시 흐름은 완료에 가깝고, 일반 만세력/차샘만세력/공유 OG의 핵심 동선은 한 번 마감된 상태다.
- 상단 compact 툴바에 전날/다음날 이동 버튼을 추가했고, 성별/양음력 선택은 변경 즉시 다시 계산되도록 맞췄다.
- 생성 버튼은 현재 폼을 제출하고, 공유 버튼도 최신 폼 값 기준 URL로 먼저 주소를 동기화한 뒤 공유하도록 맞췄다.
- 존재하지 않는 윤달 날짜는 일반 음력으로 조용히 보정하지 않고 에러로 막도록 검증을 보강했다.
- 만세력 보드의 윤달 표시는 작은 쿨톤 배지로 강조해 인지성을 높이는 방향으로 다듬고 있다.
- 현재는 이 상단 탐색 UI의 사용감을 다듬고, 후속 확장 우선순위를 정리하는 단계다.

## 다음 작업 후보
- OG 이미지에 6주와 대운을 함께 표시하는 기능을 최우선으로 추가한다.
- 기본은 보기창으로 두고, FAB 편집 버튼을 누르면 편집창이 열리는 형태로 화면을 분리하며, 보기화면 중심 탐색 UI로 확장한다.
- 상단 툴바의 정사각형 버튼 밀도, 아이콘 시각 무게, 양/음 전환 즉시 반영 체감이 모바일에서도 충분히 좋은지 추가로 다듬는다.
- 윤달 배지의 색/대비와 음력 요약 가독성이 실제 사용 화면에서 충분히 잘 보이는지 추가로 다듬는다.
- 사주 8자를 기준으로 가능한 생년월일시를 역으로 탐색하는 기능을 추가한다.
- 윤달 입력과 역사 연도 양음력 변환 기준이 기대한 한국 기준과 일치하는지 확인하고, 대표 케이스를 회귀 테스트로 보강한다.
- 음양력 변환 계산 로직의 적용 입력 범위를 `-59년 02월 13일 ~ 2050년 12월 31일`로 맞출 수 있는지 검토하고, 필요하면 변환 라이브러리 교체나 보완 데이터 전략을 정리한다.
- 차샘 명식자료를 가져오는 기능은 필요하지만, 소셜 로그인 기능을 먼저 추가한 뒤 후속 단계에서 진행한다.
## 보충 메모
- 모바일에서 새 대운/세운 즉시 선택 반응이 충분히 잘 느껴지는지 확인하고, 필요할 때만 선택 상태의 시각 강조를 더 보강한다.
