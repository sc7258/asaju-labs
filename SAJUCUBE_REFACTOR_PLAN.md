# ♻️ 사주큐브 리팩토링 및 모노레포 통합 계획 (Migration Plan)

본 문서는 기존에 단일 프로젝트로 개발되었던 **'사주큐브(Sajucube) 앱'**에서 로직과 UI를 분리하여, 새로운 `asaju-labs` 모노레포 아키텍처로 통합하기 위한 구체적인 아이디어와 작업 절차를 정리합니다.

## 🎯 핵심 목표 (Core Objective)
기존 사주큐브 앱에 결합되어 있는 **1) 만세력 계산 로직**과 **2) 사주 UI 컴포넌트**를 완벽하게 분리(Decoupling)하여 독립적인 패키지로 만들고, 이를 조립하여 **사주덱스(Sajudex)**와 새로운 **사주큐브(Sajucube)**를 구현합니다.

---

## 🧩 1단계: 의존성 분리 및 패키지화 (Packages)

가장 먼저 수행해야 할 작업으로, 기존 앱에서 공통 자산을 추출합니다.

### 1-1. `packages/saju-core` (Headless 사주 엔진)
화면(View)에 대한 의존성이 0%인 순수 TypeScript 비즈니스 로직입니다.
- **추출 대상:** 생년월일시 입력 -> 천간/지지 도출 알고리즘, 대운/세운 계산기, 신살/합충파해 로직
- **목적:** 사주덱스의 데이터 수집기(`sajudex-crawler`)에서 인물 사주를 계산할 때나, 사주큐브 웹 화면에서 실시간으로 명식을 뽑아낼 때 **동일한 엔진**을 사용하기 위함입니다.
- **테스트:** Jest나 Vitest를 이용해 특정 생년월일을 넣었을 때 정확한 8글자가 떨어지는지 Unit Test를 반드시 작성합니다.

### 1-2. `packages/ui` (공통 사주 UI 시스템)
사주와 관련된 시각적 요소(Dumb Components) 모음입니다.
- **추출 대상:** 만세력 8글자 렌더링 박스(PillarBox), 오행별 색상 테마 가이드, 대운/세운 타임라인 UI 등
- **목적:** 사주덱스와 사주큐브가 완전히 똑같은 일관된 디자인(사주 명식표 등)을 갖도록 강제합니다.

---

## 🏗️ 2단계: 신규 서비스 조립 (Apps)

분리된 패키지(`saju-core`, `ui`)를 NPM 모듈처럼 Import하여 각 서비스 앱을 재구축합니다.

### 2-1. `apps/sajucube-web` (사주큐브 리뉴얼)
기존에 만든 사주큐브 앱을 모노레포 구조에 맞게 재탄생시킵니다.
- **역할:** 사주 6개 판 정밀 분석 등 '도구(Tool)'로서의 기능에 집중합니다.
- **구조:** `saju-core`로 명식을 계산하고 `packages/ui`로 화면에 그립니다.
- **특징:** 모노레포로 들어오면서 성능 최적화와 함께 Next.js App Router 등의 최신 스택으로 업그레이드합니다.

### 2-2. `apps/sajudex-web` (사주덱스 신규 구축)
사주큐브의 로직을 활용해 새롭게 만드는 백과사전/인물 검색 서비스입니다.
- **역할:** 유명인/지인 사주 DB를 기반으로 한 일주, 조합 검색 및 명리학적 관계망 시각화
- **구조:** 백엔드/수집기에서 `saju-core`를 이용해 DB를 적재하고, 프론트엔드에서는 DB 내용과 `packages/ui`를 결합하여 인물 카드를 렌더링합니다.
- **시너지:** 사주덱스에서 특정 인물을 검색하다가 "더 정밀하게 분석하기"를 누르면, 해당 명식 데이터가 `sajucube-web`으로 매끄럽게 넘어가도록 연동합니다.

---

## 🚀 기대 효과
1. **DRY (Don't Repeat Yourself):** 만세력 계산 로직을 단 한 곳(`saju-core`)에서만 관리하므로, 윤달이나 썸머타임 등 특정 예외 케이스 버그 수정 시 모든 서비스(사주덱스, 사주큐브)에 일괄 적용됩니다.
2. **개발 속도 향상:** `ui` 컴포넌트가 이미 존재하므로, 사주덱스 화면을 만들 때 레고 블록 조립하듯 빠르게 개발할 수 있습니다.
